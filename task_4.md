## Задание 4. Логи и логирование

#### Введение

Наша комада тестирует мобильное приложение онлайн-банка. При попытке просмотреть историю операций за последний месяц приложение аварийно завершает работу.

#### Важные уровни логов для анализа проблемы

Для анализа данной проблемы наиболее важны следующие уровни логирования:

- ERROR  - основной уровень для анализа, так как именно на нём фиксируются критические ошибки, из-за которых приложение может упасть (crash).
- WARN - может содержать предупреждения, которые не приводят к падению напрямую, но указывают на потенциальную проблему (например, проблемы с данными или сетью).
- INFO - помогает понять, какие действия выполняло приложение до падения (запрос истории операций, обработка ответа и т.д.).
- DEBUG - полезен для детального анализа логики работы приложения (парсинг данных, работа с датами, фильтрация операций).

Так как приложение аварийно завершается, ключевыми являются ERROR и WARN, а INFO и DEBUG помогают восстановить цепочку действий перед сбоем.

#### Механизм получения логов

Для получения логов будет использован - Android Logcat.

**Базовые шаги:**

- Подключить тестовое устройство или эмулятор к компьютеру;
- Открыть Android Studio;
- Запустить приложение онлайн-банка на устройстве;
- Открыть вкладку Logcat;
- Очистить логи перед началом теста;
- В приложении попытаться открыть историю операций за последний месяц;
- Зафиксировать момент падения приложения;
- Скопировать логи, относящиеся к моменту crash.

#### Фильтры в Logcat для быстрого поиска причины

Для быстрого поиска причины проблемы будут применены следующие фильтры:

**По уровню логов:**

- `level:ERROR`
- `level:WARN`

**По ключевым словам:**

- `Exception`
- `Fatal`
- `Crash`
- `NullPointerException`
- `IndexOutOfBoundsException`

**По тегам (пример):**

- `TransactionHistory`
- `HistoryFragment`
- `BankApi`

**По процессу приложения:**

- `package: com.bank.app`

**Пример комбинированного фильтра:**

- `package:com.bank.app level:ERROR Exception`

**Заголовок:**

Приложение падает при открытии истории операций за последний месяц на Android и iOS.   

**Критичность (Criticality):**

- High (Высокая)

**Серьёзность (Severity):**

- Critical (Критическая)

**Окружение (Environment):**

- ОС: Android 14, iOS 17
- Устройства: Realme 9 Pro +, iPhone 16
- Версия приложения:
    - Android: 1.4.2
    - iOS: 1.4.1
- Тип сборки: Release

**Предусловия:**

- Пользователь авторизован в приложении;
- У пользователя есть операции за последний месяц.

**Шаги воспроизведения:**

- Открыть приложение онлайн-банка;
- Авторизоваться с валидными данными;
- Перейти в раздел "История операций";
- Выбрать период "Последний месяц".

**Ожидаемый результат:**

Отображается список операций за выбранный период - последний месяц.

**Фактический результат:**

Приложение аварийно завершается (crash) после выбора периода - последний месяц.

**Частота воспроизведения**

100%(воспроизводится при каждой попытке).

**Вложения:**

**Комментарий:**

Предположительно, при загрузке истории операций за последний месяц приложение получает null или пустой список от API, который не обрабатывается корректно, что приводит к падению приложения.

Проблема воспроизводится на нескольких устройствах и платформах, что свидетельствует об общей логической ошибке обработки ответа API.
