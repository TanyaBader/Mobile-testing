## Задание 5. Charles Proxy

| Функциональность сниффера | Назначение | Как тестировщик использует |
|---------------------------|------------|----------------------------|
| Фильтрация запросов | Показывает только нужные сетевые запросы и скрывает лишние | Быстро найти нужный API-запрос;<br> Убрать аналитику и системный трафик;<br> Упростить анализ. |
| Работа с логами | Сохраняет сетевой трафик в файл и позволяет открыть его позже | Приложить логи к баг-репорту;<br> Передать разработчику;<br> Разобрать проблему после тестирования. |
| Работа с зашифрованным трафиком | Позволяет видеть содержимое HTTPS-запросов | Проверить данные запроса и ответа;<br> Посмотреть headers и body;<br> Убедиться, что данные передаются корректно. |
| Map Remote | Подменяет ответ сервера на локальный или тестовый | Проверить работу приложения без сервера;<br> Эмулировать ошибки сервера;<br> Тестировать UI при разных ответах. |
| Breakpoint | Останавливает запрос или ответ, чтобы изменить его вручную | Проверить негативные сценарии;<br> Изменить данные "на лету";<br> Посмотреть реакцию приложения. |
| Rewrite | Автоматически меняет запросы или ответы по правилам | Массово менять данные;<br> Проверять разные сценарии;<br> Не менять данные вручную каждый раз. |
| Throttling | Замедляет интернет соединение | Проверить работу приложения при медленном интернете;<br> Тестировать загрузки и таймауты. |
